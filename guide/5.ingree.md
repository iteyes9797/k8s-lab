**요약:**
마스터 가이드의 마지막 퍼즐인 **Stage 5: Ingress 규칙 적용** 단계를 상세히 정리해 드립니다. 이 단계에서는 복잡한 포트 번호 대신 `nip.io`라는 무료 서비스를 활용해 **`argocd.공인IP.nip.io`**라는 깔끔한 도메인으로 접속 환경을 완성합니다. (사실 확인 완료: `nip.io`는 별도의 DNS 설정 없이 IP 주소를 호스트 이름에 포함하면 해당 IP로 라우팅해주는 와일드카드 DNS 서비스이며, Ingress 설정을 통해 80번 포트로 들어오는 트래픽을 ArgoCD 서비스로 정확히 전달할 수 있습니다.)

## 🌐 Stage 5: Ingress 규칙 적용 및 최종 접속

이 단계는 앞서 설치한 **Ingress-Nginx(Stage 4)**라는 '안내 데스크'에 **"이 주소로 오는 손님은 ArgoCD로 보내줘"**라는 명단을 등록하는 과정입니다.

### 5-1. `nip.io`란 무엇인가요?

우리는 도메인(예: `naver.com`)을 구매하지 않았습니다. 하지만 **`nip.io`**를 사용하면 본인의 공인 IP 뒤에 `.nip.io`만 붙여도 인터넷에서 유효한 도메인 주소처럼 사용할 수 있습니다.

* **예시:** 본인 IP가 `20.196.204.108`이라면, `argocd.20.196.204.108.nip.io` 주소는 전 세계 어디서든 본인의 Azure 서버로 연결됩니다.

### 5-2. Ingress 설정 파일 생성 (`argocd-ingress.yaml`)

마스터 서버 터미널에서 아래 내용을 복사하여 파일을 만듭니다.

```bash
cat <<EOF > argocd-ingress.yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: argocd-server-ingress
  namespace: argo
  annotations:
    # [🔥핵심] 아르고CD 서버가 HTTP(Insecure) 모드임을 인그레스에게 알림
    nginx.ingress.kubernetes.io/backend-protocol: "HTTP"
spec:
  ingressClassName: nginx
  rules:
  - host: argocd.20.196.204.108.nip.io # [중요] 반드시 본인의 공인 IP로 수정하세요!
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: argocd-server
            port:
              number: 80
EOF

```

### 5-3. 규칙 적용 및 확인

파일을 저장했다면 K8s 클러스터에 규칙을 적용합니다.

```bash
# 1. 인그레스 규칙 적용
kubectl apply -f argocd-ingress.yaml

# 2. 적용 상태 확인 (ADDRESS 부분에 공인 IP가 나오면 성공)
kubectl get ingress -n argo

```

### 5-4. 최종 접속 테스트 (축하합니다! 🎉)

이제 더 이상 포트 번호를 붙이지 마세요.

1. **브라우저 접속:** 시크릿 창을 열고 주소창에 아래와 같이 입력합니다.
👉 **`http://argocd.20.196.204.108.nip.io`**
2. **로그인 정보:**
* **ID:** `admin`
* **PW 확인:** `kubectl -n argo get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d; echo`

## 🚨 Stage 5의 "피와 살이 되는" 삽질 포인트 정리

### 📍 삽질 1: "IP 주소 오타"

* **증상:** 분명히 인그레스를 적용했는데 '사이트에 연결할 수 없음'이 뜸.
* **원인:** `host:` 항목에 적은 IP 주소가 실제 Azure 공인 IP와 점 하나라도 다를 경우 `nip.io`가 길을 찾지 못합니다.
* **해결:** `kubectl get nodes -o wide`나 Azure 포털에서 공인 IP를 다시 한번 확인하고 오타를 수정하세요.

### 📍 삽질 2: "Annotation 누락"

* **증상:** 접속은 되는데 `502 Bad Gateway`가 뜸.
* **원인:** `nginx.ingress.kubernetes.io/backend-protocol: "HTTP"` 설정이 없으면, 인그레스가 아르고CD에게 HTTPS로 말을 걸려고 시도하여 통신이 끊깁니다.
* **해결:** 위에 드린 YAML 코드의 `annotations` 부분을 빠짐없이 복사했는지 확인하세요.

### 📍 삽질 3: "DNS 전파 대기"

* **증상:** 모든 게 맞는데 아주 가끔 접속이 안 됨.
* **해결:** `nip.io`는 실시간 서비스이지만, 네트워크 환경에 따라 1~2분 정도 대기가 필요할 수 있습니다. 잠시 후 새로고침 해보세요.
