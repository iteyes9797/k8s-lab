# 🛠️ Stage 1: Terraform - Azure 인프라 구축 "끝장판"

이 가이드는 로컬 PC(Windows/Mac)에서 Azure에 K8s용 서버 뼈대를 만드는 전 과정을 다룹니다.

## 1-1. 로컬 환경에 Terraform 설치 (처음부터 시작하기)

만약 테라폼이 안 깔려 있다면 이것부터 하셔야 합니다.

1. **다운로드:** [Terraform 공식 홈페이지](https://www.terraform.io/downloads)에서 OS에 맞는 바이너리 다운로드.
2. **환경 변수:** `terraform.exe`가 있는 폴더를 시스템 환경 변수 `Path`에 등록.
3. **확인:** 터미널에서 `terraform -v`를 쳤을 때 버전이 나오면 성공.
4. **Azure CLI 설치:** `az login`을 위해 [Azure CLI](https://learn.microsoft.com/ko-kr/cli/azure/install-azure-cli)도 반드시 설치되어 있어야 합니다.

## 1-2. 전체 코드: `main.tf` (복사해서 바로 사용)

작업 폴더를 하나 만드시고(예: `C:\k8s-infra`), 그 안에 `main.tf` 파일을 만들어 아래 내용을 통째로 붙여넣으세요. 오늘 우리가 해결한 **IP Forwarding**과 **NodePort NSG**가 완벽히 녹아 있습니다.

```hcl
# 1. Provider 설정
terraform {
  required_providers {
    azurerm = {
      source  = "hashicorp/azurerm"
      version = "~> 3.0"
    }
  }
}

provider "azurerm" {
  features {}
}

# 2. 리소스 그룹 생성
resource "azurerm_resource_group" "rg" {
  name     = "k8s-resource-group"
  location = "Korea Central" #
}

# 3. 가상 네트워크 및 서브넷 생성
resource "azurerm_virtual_network" "vnet" {
  name                = "k8s-vnet"
  address_space       = ["192.168.0.0/16"]
  location            = azurerm_resource_group.rg.location
  resource_group_name = azurerm_resource_group.rg.name
}

resource "azurerm_subnet" "subnet" {
  name                 = "hw-subnet" #
  resource_group_name  = azurerm_resource_group.rg.name
  virtual_network_name = azurerm_virtual_network.vnet.name
  address_prefixes     = ["192.168.0.0/24"]
}

# 4. 공인 IP 생성 (마스터 서버 접속용)
resource "azurerm_public_ip" "pip" {
  name                = "k8s-master-pip"
  location            = azurerm_resource_group.rg.location
  resource_group_name = azurerm_resource_group.rg.name
  allocation_method   = "Static"
}

# 5. [중요!] 네트워크 보안 그룹 (NSG) - 여기서 우리가 삽질했었죠!
resource "azurerm_network_security_group" "nsg" {
  name                = "k8s-nsg" #
  location            = azurerm_resource_group.rg.location
  resource_group_name = azurerm_resource_group.rg.name

  # SSH 허용 (기본)
  security_rule {
    name                       = "SSH"
    priority                   = 1001
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "22"
    source_address_prefix      = "*"
    destination_address_prefix = "*"
  }

  # [🔥삽질 포인트 1] ArgoCD NodePort 대역 전체 허용
  security_rule {
    name                       = "Allow-NodePort" #
    priority                   = 1011
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "30000-32767"
    source_address_prefix      = "*"
    destination_address_prefix = "*"
  }
}

# 6. [핵심!!] 네트워크 인터페이스 (NIC) - IP Forwarding 설정
resource "azurerm_network_interface" "nic" {
  name                = "k8s-master01-nic" #
  location            = azurerm_resource_group.rg.location
  resource_group_name = azurerm_resource_group.rg.name
  
  # [🔥삽질 포인트 2] 이 옵션이 꺼져있으면 파드 통신이 드랍됩니다.
  enable_ip_forwarding = true #

  ip_configuration {
    name                          = "internal"
    subnet_id                     = azurerm_subnet.subnet.id
    private_ip_address_allocation = "Static"
    private_ip_address            = "192.168.0.106" #
    public_ip_address_id          = azurerm_public_ip.pip.id
  }
}

# 7. NIC와 NSG 연결
resource "azurerm_network_interface_security_group_association" "asoc" {
  network_interface_id      = azurerm_network_interface.nic.id
  network_security_group_id = azurerm_network_security_group.nsg.id
}

# 8. 가상 머신 (VM) 생성
resource "azurerm_linux_virtual_machine" "vm" {
  name                = "k8s-master01" #
  resource_group_name = azurerm_resource_group.rg.name
  location            = azurerm_resource_group.rg.location
  size                = "Standard_DS2_v2" # 스터디용 적정 사양
  admin_username      = "azureuser"
  network_interface_ids = [
    azurerm_network_interface.nic.id,
  ]

  admin_ssh_key {
    username   = "azureuser"
    public_key = file("~/.ssh/id_rsa.pub") # 본인의 공개키 경로
  }

  os_disk {
    caching              = "ReadWrite"
    storage_account_type = "Standard_LRS"
  }

  source_image_reference {
    publisher = "Canonical"
    offer     = "0001-com-ubuntu-server-focal"
    sku       = "20_04-lts"
    version   = "latest"
  }
}

```

## 1-3. "피가 되고 살이 되는" 삽질 정리 (Deep-Dive)

### 🚨 삽질 1: "NSG는 서브넷에만 거는 게 아니다!"

* **증상:** Azure 포털에서 `k8s-nsg`를 만들고 30000번 포트를 열었는데도 접속이 안 됨.
* **원인:** NSG가 서브넷에는 걸려있지만, VM의 **네트워크 인터페이스(NIC)** 레벨에서 한 번 더 막고 있을 수 있음.
* **해결:** 위 코드처럼 `azurerm_network_interface_security_group_association`를 사용해 NIC에 직접 NSG를 꽝! 박아줘야 확실합니다.

### 🚨 삽질 2: "클라우드의 자물쇠, IP Forwarding"

* **증상:** `port-forward`를 해도 `connection reset`이 뜨고, 내부 통신이 자꾸 끊김.
* **원인:** Azure 하이퍼바이저는 보안상 자기 VM IP가 아닌 패킷(파드 IP)이 랜카드를 통과하려고 하면 해킹으로 간주하고 버립니다.
* **해결:** `enable_ip_forwarding = true` 설정을 통해 "이 서버는 K8s 노드니까 다른 IP 패킷도 통과시켜줘!"라고 허락을 받아야 합니다.

### 🚨 삽질 3: "Public IP 할당 방식"

* **증상:** 서버를 껐다 켜면 IP가 바뀌어서 브라우저 접속 주소를 매번 수정해야 함.
* **원인:** Public IP 설정을 `Dynamic`으로 했기 때문.
* **해결:** `allocation_method = "Static"`으로 고정해서 스터디 내내 같은 IP를 쓰도록 만듭니다.

## 1-4. 배포 명령어 (로컬 PC 터미널에서 실행)

```bash
# 1. Azure 로그인
az login

# 2. 테라폼 초기화 (플러그인 다운로드)
terraform init

# 3. 변경 사항 미리보기 (실수 방지)
terraform plan

# 4. 실제 인프라 생성 (Yes 입력)
terraform apply

```
