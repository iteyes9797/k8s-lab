# 📦 Stage 2: Ansible & Helm - 서버 설정 및 ArgoCD 배포 "완성판"

테라폼으로 만든 가상 머신(VM)에 접속할 준비가 되었다면, 이제 로컬 PC에서 앤서블을 통해 서버 환경을 잡고 아르고CD 설정을 배달할 차례입니다.

## 2-1. 로컬 PC 작업 디렉토리 구조

먼저 로컬 PC의 앤서블 작업 폴더를 아래와 같이 구성하세요. 경로가 꼬이면 파일 배달이 안 되므로 구조를 똑같이 맞추는 것이 중요합니다.

```text
ansible-k8s/
├── hosts.ini              # 접속할 서버 IP 정보
├── setup_argocd.yml       # 실행할 플레이북
└── files/
    └── argocd-values.yaml # 아르고CD 상세 설정 파일

```

---

## 2-2. 상세 파일 내용 (복사해서 사용)

### 1. `hosts.ini` (서버 주소록)

테라폼에서 생성된 공인 IP 주소를 여기에 적으세요.

```ini
[master]
k8s-master01 ansible_host=<마스터_공인_IP> ansible_user=azureuser ansible_ssh_private_key_file=~/.ssh/id_rsa

```

### 2. `files/argocd-values.yaml` (아르고CD 성격 정의)

우리가 가장 고생했던 **프로토콜(HTTPS) 충돌**을 방지하기 위해 설치 시점부터 보안 옵션을 조정합니다.

```yaml
# argocd-values.yaml
server:
  extraArgs:
    - --insecure             # [🔥핵심] 내부적으로 HTTP 통신을 허용하여 접속 오류 차단
  service:
    type: NodePort           # 설치와 동시에 외부 포트를 엽니다.
    nodePort:
      http: 30008            # 포트를 수동으로 지정하면 관리가 훨씬 쉽습니다.
      https: 30043

```

### 3. `setup_argocd.yml` (자동화 시나리오)

서버에 접속해서 폴더를 만들고 설정을 배달하는 전체 과정입니다.

```yaml
- name: ArgoCD 배포 환경 구축
  hosts: master
  become: yes # [🔥중요] root 권한으로 실행하여 권한 문제 방지
  tasks:
    - name: 1. 헬름 차트 보관용 폴더 생성
      file:
        path: /home/azureuser/helm_charts
        state: directory
        owner: azureuser
        group: azureuser
        mode: '0755'

    - name: 2. 로컬의 설정 파일을 서버로 배달
      copy:
        src: files/argocd-values.yaml
        dest: /home/azureuser/helm_charts/argocd-values.yaml
        owner: azureuser
        group: azureuser

    - name: 3. (선택) Helm 설치 여부 확인
      command: helm version
      register: helm_check
      ignore_errors: yes

    - name: 4. Helm이 없다면 경고 메시지 출력
      debug:
        msg: "서버에 Helm이 설치되어 있지 않습니다. 먼저 Helm을 설치해주세요!"
      when: helm_check.failed

```

## 2-3. "뼈아픈" 삽질 정리 (Deep-Dive)

앤서블 단계에서 우리가 겪었던 실제 문제들을 해결책과 함께 정리했습니다.

### 🚨 삽질 1: "파일은 배달됐는데, 왜 못 찾지?"

* **상황:** 앤서블로 파일을 보냈는데, 서버에서 `ls`를 쳐도 파일이 안 보임.
* **원인:** 앤서블 `become: yes`로 파일을 만들면 소유권이 `root`가 되어 `azureuser` 계정에서 접근이 제한되거나, 경로가 `/root/` 밑으로 들어가는 경우가 발생합니다.
* **해결:** 위 코드처럼 `owner: azureuser`를 명시하고, 절대 경로인 `/home/azureuser/helm_charts`를 사용하여 경로 이탈을 방지했습니다.

### 🚨 삽질 2: "Values 파일이 적용이 안 돼요!"

* **상황:** `helm install`을 했는데 설정한 포트(30008)가 아니라 랜덤 포트가 잡힘.
* **원인:** 설치 명령어 실행 시 `-f argocd-values.yaml` 경로를 상대 경로로 적었는데, 현재 터미널 위치와 파일 위치가 달랐기 때문입니다.
* **해결:** 반드시 `cd /home/azureuser/helm_charts`로 이동한 뒤 설치를 진행하거나, 전체 경로를 다 적어줘야 합니다.

## 2-4. 실행 명령어

로컬 PC 터미널(앤서블 폴더 내)에서 아래 명령어를 실행하세요.

```bash
# 1. 접속 테스트 (서버와 통신이 되는지 확인)
ansible master -m ping -i hosts.ini

# 2. 플레이북 실행 (설정 파일 배달)
ansible-playbook -i hosts.ini setup_argocd.yml

```
