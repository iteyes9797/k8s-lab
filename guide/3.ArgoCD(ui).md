# 🐙 Stage 3: Azure VM 기반 ArgoCD 운영 표준 구축 가이드

## 1단계: 테라폼 (인프라 정문 개방)

로컬 PC의 `network.tf`에서 30000번대 포트 대신 **80(HTTP)과 443(HTTPS)**을 엽니다.

* **핵심 코드:**
```hcl
resource "azurerm_network_security_rule" "web_traffic" {
  name                       = "Allow-Web"
  priority                   = 1005
  direction                  = "Inbound"
  access                     = "Allow"
  protocol                   = "Tcp"
  source_port_range          = "*"
  destination_port_ranges    = ["80", "443"] # 80/443만 열면 끝!
  source_address_prefix      = "*"
  destination_address_prefix = "*"
  resource_group_name        = azurerm_resource_group.rg.name
  network_security_group_name = azurerm_network_security_group.nsg.name
}

```


* **삽질 방지:** `enable_ip_forwarding = true`는 여전히 필수입니다.

## 2단계: 앤서블 (설정 파일 배달)

서버에 `argocd-values.yaml`을 보낼 때, 인그레스 환경에 맞게 내용을 수정합니다.

* **`files/argocd-values.yaml` 내용:**
```yaml
server:
  extraArgs:
    - --insecure # 인그레스가 TLS를 처리할 것이므로 내부적으론 HTTP 사용
  service:
    type: ClusterIP # 외부 노출은 인그레스가 할 것이니 다시 기본값으로!

```



## 3단계: 서버 작업 (ArgoCD & Ingress 설치)

마스터 서버 터미널에서 아래 순서대로 명령어를 복사해서 붙여넣으세요.

1. **ArgoCD 설치:**
```bash
helm install argocd argo/argo-cd -n argo --create-namespace -f argocd-values.yaml

```


2. **Ingress-Nginx 설치 (운영 비기):**
```bash
helm upgrade --install ingress-nginx ingress-nginx/ingress-nginx \
  --namespace ingress-nginx --create-namespace \
  --set controller.hostNetwork=true \
  --set controller.kind=DaemonSet

```


* **왜 이렇게 하나요?** 로드밸런서가 없는 VM 환경에서 인그레스가 80/443 포트를 직접 점유하게 만드는 설정입니다.



## 4단계: 접속 규칙 생성 (Ingress)

"어떤 주소로 들어오면 ArgoCD로 보내줄지" 규칙을 만듭니다.

* **`argocd-ingress.yaml` 생성 및 적용:**
```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: argocd-server-ingress
  namespace: argo
  annotations:
    nginx.ingress.kubernetes.io/backend-protocol: "HTTP"
spec:
  ingressClassName: nginx
  rules:
  - host: argocd.<공인IP>.nip.io # 예: argocd.20.196.204.108.nip.io
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: argocd-server
            port:
              number: 80

```


```bash
kubectl apply -f argocd-ingress.yaml

```



## 5단계: 최종 접속 (Finish)

* **URL:** `http://argocd.<공인IP>.nip.io`
* **비밀번호 확인:**
```bash
kubectl -n argo get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d; echo

```

### 💡 최종 정리

1. **순서:** 테라폼(인프라) → 앤서블(파일) → ArgoCD 설치(몸통) → Ingress 설치(입구) → Ingress 규칙 적용(안내판).
2. **삽질의 의미:** 아까 했던 NodePort 삽질이 있었기에, "왜 인그레스를 써야 하는지"와 "포트 번호 없는 접속이 얼마나 편한지"를 완벽하게 이해하시게 된 겁니다!
